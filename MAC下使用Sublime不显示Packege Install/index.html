MAC下使用Sublime不显示Packege Install

发表于 2024-11-08 更新于 2024-11-15

314152 / MacOS / 新版MacOS上Sublime Text 4无法使用Install Package Control或安装Install Package Control 找不到Install Package选项的解决办法
新版MacOS上Sublime Text 4无法使用Install Package Control或安装Install Package Control 找不到Install Package选项的解决办法：

最新升级了MacOS到Ventura和Sonoma,发现安装Sublime Text4之后，无法使用Install Package Control，即使安装了Install Package Control也无法正常使用Install Pakcage。确实要命啊。

解决方法：

下载：PackageControl

之后得到一个Package.Control.sublime.package文件，将文件重命名为Package Control.sublime-package
随后将这个文件放置到插件安装目录的（Installed Packages）目录下，路径为：/Users/用户名/Library/Application Support/Sublime Text/Installed Packages

这里有个坑，不是Browser Package浏览插件操作打开的目录 ，而是这个目录的上级目录。浏览插件打开的是User下的，而Installed Packages目录和User同级。需要将插件放到Installed Packages目录下。

放置好之后，重启Sublime就可以正常使用Sublime Text了。
